دوره پایتون

دوره پایتون شما را با اصول اولیه و اساسی برنامه‌نویسی با پایتون آشنا می‌کند و پایه‌ای محکم برای شما می‌سازد. دوره به گونه ای طراحی شده که برای همه دانشجویان و علاقه‌مندان به یادگیری پایتون، و حتی افرادی که هیچ پیش زمینه‌ای در برنامه‌نویسی ندارند مناسب است.

برنامه درسی

  • آشنایی با مفاهیم اولیه برنامه‌نویسی و پایتون
  • کار با متغیرها و انواع داده‌ها
  • کنترل جریان برنامه با شرط‌ها و حلقه‌ها
  • کار با لیست‌ها، دیکشنری‌ها و ساختارهای داده‌ها
  • تعریف از توابع، شیءگرایی و مدیریت بهتر کدها

جزئیات دوره

  • آموزش گام‌به‌گام، ساده و قابل فهم
  • تمرین‌های عملی برای تسلط بر مفاهیم پایه
  • دسترسی به مطالب و دروس در هر زمان

پس از اتمام این دوره، شما می‌توانید:

  • برنامه‌های پایتونی بنویسید و اجرا کنید
  • مفاهیم برنامه‌نویسی را درک و اعمال کنید
  • آمادگی برای یادگیری مفاهیم مبنا بر حوزه انتخاب شده، یادگیری کتابخانه‌های پایتون وغیره.

پایتون چیست؟

پایتون یک زبان برنامه‌نویسی تفسیر شده و سطح بالا است که توسط Guido van Rossum نوشته شده و در سال 1991 منتشر شد.

موارد استفاده پایتون:

  • توسعه وب (سمت سرور)
  • توسعه نرم‌افزار
  • علم و تحلیل داده
  • ماشین لرنینگ
  • محاسبات ریاضی
  • اسکریپت‌نویسی
  • اتوماسیون
  • توسعه و ساخت بازی

در اصل، تطبیق‌پذیری و کتابخانه‌های گسترده پایتون، آن را به زبان و ابزاری ارزشمند تبدیل کرده است.

پایتون چه کارهایی می‌تواند انجام دهد؟

  • پایتون می‌تواند در سرور برای ایجاد برنامه‌های تحت وب استفاده شود.
  • پایتون می‌تواند همراه با نرم‌افزارها برای ایجاد گردش کار استفاده شود.
  • پایتون می‌تواند به سیستم‌های پایگاه داده متصل شود، فایل‌ها را بخواند و تغییر دهد.
  • پایتون می‌تواند برای پردازش داده‌های حجیم و انجام محاسبات پیچیده ریاضی استفاده شود.
  • پایتون می‌تواند برای نمونه‌سازی سریع یا توسعه نرم‌افزارهای آماده تولید استفاده شود.

چرا پایتون؟

  • پایتون روی پلتفرم‌های مختلف کار می‌کند (ویندوز، مک، لینوکس، رزبری پای و غیره).
  • پایتون دارای سینتکسی ساده و شبیه به زبان انگلیسی است.
  • سینتکس پایتون به توسعه‌دهندگان اجازه می‌دهد برنامه‌هایی با خطوط کد کمتر نسبت به برخی زبان‌های دیگر بنویسند.
  • پایتون روی یک سیستم مفسر اجرا می‌شود، به این معنی که کد بلافاصله پس از نوشتن قابل اجرا است. این ویژگی باعث می‌شود نمونه‌سازی بسیار سریع انجام شود.
  • پایتون را می‌توان به صورت رویه‌ای، شیءگرا یا تابعی استفاده کرد.

نکات مهم

جدیدترین نسخه اصلی پایتون، Python 3 است که در این آموزش از آن استفاده خواهیم کرد. با این حال، Python 2 اگرچه فقط به‌روزرسانی‌های امنیتی دریافت می‌کند، هنوز هم نسبتاً محبوب است.

در این آموزش، کدهای پایتون در کد ادیتور وبسایت ما قابل نوشتن است. همچنین می‌توان پایتون را در محیط‌های توسعه یکپارچه (IDE) یا کد ادیتور های مانند Pycharm, VSCode و غیره نوشت که به ویژه هنگام مدیریت مجموعه‌های بزرگتری از فایل‌های پایتون مفید هستند.

مقایسه سینتکس پایتون با سایر زبان‌های برنامه‌نویسی

پایتون برای خوانایی طراحی شده و شباهت‌هایی با زبان انگلیسی با تأثیر از ریاضیات دارد.

پایتون از خطوط جدید برای تکمیل یک دستور استفاده می‌کند، برخلاف بسیاری از زبان‌های برنامه‌نویسی دیگر که معمولاً از نقطه‌ویرگول یا پرانتز استفاده می‌کنند.

پایتون برای تعریف محدوده (scope) از تورفتگی (indentation) و فاصله استفاده می‌کند؛ مانند محدوده حلقه‌ها، توابع و کلاس‌ها. بسیاری از زبان‌های برنامه‌نویسی دیگر برای این منظور از آکولاد استفاده می‌کنند.

قوانین نام‌گذاری متغیرها

متغیرها می‌توانند نامی کوتاه (مانند x و y) یا نامی توصیفی‌تر (مانند age، carname، total_volume) داشته باشند.

  • نام متغیر باید با یک حرف یا زیرخط (_) شروع شود.
  • نام متغیر نمی‌تواند با عدد شروع شود.
  • نام متغیر فقط می‌تواند شامل حروف، اعداد و زیرخط باشد (A-z، 0-9، _).
  • نام‌های متغیر به حروف کوچک و بزرگ حساس هستند (age، Age و AGE سه متغیر متفاوت هستند).
  • نام متغیر نمی‌تواند یکی از کلمات کلیدی پایتون باشد.

نمونه‌هایی از نام‌های غیرقانونی:

illegal_names.py
SyntaxError: invalid syntax

نام‌های چندکلمه‌ای

نام‌های متغیر با بیش از یک کلمه ممکن است خواندن را دشوار کنند. تکنیک‌هایی برای بهبود خوانایی وجود دارد:

  • Camel Case: هر کلمه به جز اولی با حرف بزرگ شروع می‌شود: myVariableName
  • Pascal Case: هر کلمه با حرف بزرگ شروع می‌شود: MyVariableName
  • Snake Case: کلمات با زیرخط از هم جدا می‌شوند: my_variable_name

تبدیل نوع (Casting) در پایتون

ممکن است گاهی بخواهید نوع خاصی را برای یک متغیر مشخص کنید. این کار با استفاده از تبدیل نوع (Casting) انجام می‌شود. پایتون یک زبان شیءگرا است و از کلاس‌ها برای تعریف انواع داده‌ها، از جمله انواع اولیه، استفاده می‌کند.

تبدیل نوع در پایتون با استفاده از توابع سازنده (constructor functions) زیر انجام می‌شود:

  • int()- یک عدد صحیح را از یک مقدار عددی صحیح، یک مقدار اعشاری (با حذف اعشار)، یا یک رشته (به شرطی که رشته یک عدد کامل را نشان دهد) می‌سازد.

  • str() - یک رشته را از انواع داده‌های مختلف، از جمله رشته‌ها، اعداد صحیح و اعداد اعشاری می‌سازد.

متدهای رشته - strings

پایتون مجموعه‌ای از متدهای داخلی برای رشته‌ها دارد. توجه داشته باشید که تمام متدهای رشته مقادیر جدیدی را برمی‌گردانند و رشته اصلی را تغییر نمی‌دهند.

لیست متدهای رشته:

  • capitalize() - اولین کاراکتر را به حروف بزرگ تبدیل می‌کند.
  • casefold() - رشته را به حروف کوچک تبدیل می‌کند.
  • center() - رشته را در مرکز قرار می‌دهد.
  • count() - تعداد دفعات وقوع یک مقدار مشخص در رشته را برمی‌گرداند.
  • encode() - نسخه کدگذاری‌شده رشته را برمی‌گرداند.
  • endswith() - اگر رشته با مقدار مشخص‌شده پایان یابد، True برمی‌گرداند.
  • expandtabs() - اندازه تب را در رشته تنظیم می‌کند.
  • find() - رشته را برای مقدار مشخص‌شده جستجو می‌کند و موقعیت آن را برمی‌گرداند.
  • format() - مقادیر مشخص‌شده را در رشته قالب‌بندی می‌کند.
  • format_map() - مقادیر مشخص‌شده را در رشته قالب‌بندی می‌کند.
  • index() - رشته را برای مقدار مشخص‌شده جستجو می‌کند و موقعیت آن را برمی‌گرداند.
  • isalnum() - اگر تمام کاراکترهای رشته الفبایی یا عددی باشند، True برمی‌گرداند.
  • isalpha() - اگر تمام کاراکترهای رشته حروف الفبا باشند، True برمی‌گرداند.
  • isascii() - اگر تمام کاراکترهای رشته اسکی باشند، True برمی‌گرداند.
  • isdecimal() - اگر تمام کاراکترهای رشته اعداد اعشاری باشند، True برمی‌گرداند.
  • isdigit() - اگر تمام کاراکترهای رشته اعداد باشند، True برمی‌گرداند.
  • isidentifier() - اگر رشته یک شناسه معتبر باشد، True برمی‌گرداند.
  • islower() - اگر تمام کاراکترهای رشته حروف کوچک باشند، True برمی‌گرداند.
  • isnumeric() - اگر تمام کاراکترهای رشته عددی باشند، True برمی‌گرداند.
  • isprintable() - اگر تمام کاراکترهای رشته قابل چاپ باشند، True برمی‌گرداند.
  • isspace() - اگر تمام کاراکترهای رشته فضای خالی باشند، True برمی‌گرداند.
  • istitle() - اگر رشته از قوانین عنوان پیروی کند، True برمی‌گرداند.
  • isupper() - اگر تمام کاراکترهای رشته حروف بزرگ باشند، True برمی‌گرداند.
  • join() - عناصر یک شیء قابل تکرار را به انتهای رشته متصل می‌کند.
  • ljust() - نسخه‌ای از رشته را که به سمت چپ تراز شده است، برمی‌گرداند.
  • lower() - رشته را به حروف کوچک تبدیل می‌کند.
  • lstrip() - نسخه‌ای از رشته را که فضای خالی از سمت چپ حذف شده است، برمی‌گرداند.
  • maketrans() - جدول ترجمه‌ای برای استفاده در ترجمه‌ها برمی‌گرداند.
  • partition() - رشته را به سه بخش تقسیم می‌کند و یک تاپل برمی‌گرداند.
  • replace() - رشته‌ای را برمی‌گرداند که یک مقدار مشخص‌شده با مقدار دیگری جایگزین شده است.
  • rfind() - رشته را برای مقدار مشخص‌شده جستجو می‌کند و آخرین موقعیت آن را برمی‌گرداند.

  • rpartition() - رشته را به سه بخش تقسیم می‌کند و یک تاپل برمی‌گرداند.

  • split() - رشته را در جداکننده مشخص‌شده تقسیم می‌کند و یک لیست برمی‌گرداند.

  • strip() - نسخه‌ای از رشته را که فضای خالی از ابتدا و انتها حذف شده است، برمی‌گرداند.
  • swapcase() - حروف کوچک را به بزرگ و بالعکس تبدیل می‌کند.

  • translate() - یک رشته ترجمه‌شده برمی‌گرداند.

لیست‌ها در پایتون

لیست‌ها برای ذخیره چندین آیتم در یک متغیر واحد استفاده می‌شوند.

لیست‌ها یکی از چهار نوع داده داخلی در پایتون برای ذخیره مجموعه‌های داده هستند. سه نوع دیگر تاپل، مجموعه و دیکشنری هستند که هر کدام ویژگی‌ها و کاربردهای متفاوتی دارند.

آیتم‌های لیست

آیتم‌های لیست مرتب، قابل تغییر و اجازه‌دهنده مقادیر تکراری هستند.

آیتم‌های لیست دارای اندیس هستند، اولین آیتم اندیس [0]، دومین آیتم اندیس [1] و غیره دارد.

مرتب

وقتی می‌گوییم لیست‌ها مرتب هستند، یعنی آیتم‌ها ترتیب مشخصی دارند و این ترتیب تغییر نمی‌کند. آیتم‌های جدید در انتهای لیست قرار می‌گیرند.

نکته: برخی متدهای لیست ترتیب را تغییر می‌دهند، اما به طور کلی ترتیب آیتم‌ها ثابت است.

قابل تغییر

لیست قابل تغییر است، یعنی می‌توانیم آیتم‌ها را پس از ایجاد لیست تغییر دهیم، اضافه کنیم یا حذف کنیم.

متدهای لیست

پایتون مجموعه‌ای از متدهای داخلی برای کار با لیست‌ها دارد:

متد توضیح
append() یک عنصر را به انتهای لیست اضافه می‌کند
clear() تمام عناصر را از لیست حذف می‌کند
copy() یک کپی از لیست برمی‌گرداند
count() تعداد عناصر با مقدار مشخص‌شده را برمی‌گرداند
extend() عناصر یک لیست (یا هر شیء قابل تکرار) را به انتهای لیست فعلی اضافه می‌کند
index() اندیس اولین عنصر با مقدار مشخص‌شده را برمی‌گرداند
insert() یک عنصر را در موقعیت مشخص‌شده اضافه می‌کند
pop() عنصر در موقعیت مشخص‌شده را حذف می‌کند
remove() آیتم با مقدار مشخص‌شده را حذف می‌کند
reverse() ترتیب لیست را معکوس می‌کند
sort() لیست را مرتب می‌کند

تاپل‌ها در پایتون

تاپل‌ها برای ذخیره چندین آیتم در یک متغیر واحد استفاده می‌شوند.

تاپل یکی از چهار نوع داده داخلی در پایتون برای ذخیره مجموعه‌های داده است. سه نوع دیگر لیست، مجموعه و دیکشنری هستند که هر کدام ویژگی‌ها و کاربردهای متفاوتی دارند.

تاپل مجموعه‌ای است که مرتب و غیرقابل تغییر است.

متدهای مجموعه

پایتون مجموعه‌ای از متدهای داخلی برای کار با مجموعه‌ها دارد:

متد میانبر توضیح
add() - یک عنصر به مجموعه اضافه می‌کند
clear() - تمام عناصر را از مجموعه حذف می‌کند
copy() - یک کپی از مجموعه برمی‌گرداند
difference() - مجموعه‌ای شامل تفاوت بین دو یا چند مجموعه برمی‌گرداند
difference_update() -= آیتم‌هایی که در مجموعه دیگر نیز هستند را از مجموعه حذف می‌کند
discard() - آیتم مشخص‌شده را حذف می‌کند
intersection() & مجموعه‌ای شامل تقاطع دو مجموعه برمی‌گرداند
intersection_update() &= آیتم‌هایی که در مجموعه‌های دیگر نیستند را از مجموعه حذف می‌کند
isdisjoint() - بررسی می‌کند که آیا دو مجموعه تقاطع دارند یا خیر
issubset() <= بررسی می‌کند که آیا مجموعه دیگری شامل این مجموعه است یا خیر
issuperset() >= بررسی می‌کند که آیا این مجموعه شامل مجموعه دیگری است یا خیر
pop() - یک عنصر را از مجموعه حذف می‌کند
remove() - عنصر مشخص‌شده را حذف می‌کند
symmetric_difference() ^ مجموعه‌ای با تفاوت‌های متقارن دو مجموعه برمی‌گرداند
symmetric_difference_update() ^= تفاوت‌های متقارن این مجموعه و مجموعه دیگر را درج می‌کند
union() | مجموعه‌ای شامل اتحاد مجموعه‌ها برمی‌گرداند
update() |= مجموعه را با اتحاد این مجموعه و دیگران به‌روزرسانی می‌کند

مرتب یا نامرتب؟

دستورات شرطی If ... Else در پایتون

پایتون از شرایط منطقی معمول در ریاضیات پشتیبانی می‌کند:

  • برابر: ==
  • نابرابر: !=
  • کمتر از: <
  • کمتر یا مساوی: <=
  • بیشتر از: >
  • بیشتر یا مساوی: >=

این شرایط می‌توانند به روش‌های مختلفی استفاده شوند، معمولاً در دستورات if و حلقه‌ها.

دستور if با استفاده از کلمه کلیدی if نوشته می‌شود:

در این مثال، ما از دو متغیر a و b استفاده می‌کنیم تا بررسی کنیم آیا b بزرگ‌تر از a است. از آنجا که a برابر با 33 و b برابر با 200 است، می‌دانیم که 200 بزرگ‌تر از 33 است، بنابراین پیام "b is greater than a" چاپ می‌شود.

بازگشت (Recursion)

پایتون همچنین از بازگشت تابع پشتیبانی می‌کند، به این معنا که یک تابع تعریف‌شده می‌تواند خودش را فراخوانی کند.

بازگشت یک مفهوم رایج ریاضی و برنامه‌نویسی است. این یعنی تابع خودش را فراخوانی می‌کند. این مزیت را دارد که می‌توانید از طریق داده‌ها حلقه بزنید تا به نتیجه برسید.

توسعه‌دهنده باید با بازگشت بسیار محتاط باشد، زیرا به راحتی می‌توان تابعی نوشت که هرگز متوقف نمی‌شود یا منابع زیادی از حافظه یا قدرت پردازنده مصرف می‌کند. اما وقتی به درستی نوشته شود، بازگشت می‌تواند رویکردی بسیار کارآمد و ریاضی‌گونه برای برنامه‌نویسی باشد.

در این مثال، تابع tri_recursion() تابعی است که تعریف کرده‌ایم تا خودش را فراخوانی کند ("بازگشت"). ما از متغیر k به عنوان داده استفاده می‌کنیم که هر بار که بازگشت می‌کنیم یک واحد کاهش می‌یابد (-1). بازگشت زمانی پایان می‌یابد که شرط دیگر بزرگ‌تر از 0 نباشد (یعنی وقتی 0 باشد).

برای توسعه‌دهندگان جدید، ممکن است مدتی طول بکشد تا دقیقاً متوجه شوند این چگونه کار می‌کند. بهترین راه برای فهمیدن، آزمایش و اصلاح آن است:

برنامه‌نویسی شیءگرا (OOP) در پایتون

برنامه‌نویسی شیءگرا چیست؟

OOP مخفف Object-Oriented Programming (برنامه‌نویسی شیءگرا) است.

پایتون یک زبان برنامه‌نویسی شیءگرا است که به شما امکان می‌دهد کد خود را با استفاده از کلاس‌ها و اشیاء ساختاربندی کنید تا سازمان‌دهی و قابلیت استفاده مجدد بهتری داشته باشید.

مزایای برنامه‌نویسی شیءگرا

  • ساختاری واضح به برنامه‌ها ارائه می‌دهد.
  • نگهداری، استفاده مجدد و اشکال‌زدایی کد را آسان‌تر می‌کند.
  • به حفظ اصل DRY (Don't Repeat Yourself - خودت را تکرار نکن) کمک می‌کند.
  • امکان ساخت برنامه‌های قابل استفاده مجدد با کد کمتر را فراهم می‌کند.

نکته: اصل DRY به این معناست که باید از نوشتن کد تکراری بیش از یک بار اجتناب کنید. کدهای تکراری را به توابع یا کلاس‌ها منتقل کنید و از آن‌ها مجدداً استفاده کنید.

کلاس‌ها و اشیاء چیستند؟

کلاس‌ها و اشیاء دو مفهوم اصلی در برنامه‌نویسی شیءگرا هستند.

یک کلاس تعریف می‌کند که یک شیء باید چگونه باشد، و یک شیء بر اساس آن کلاس ایجاد می‌شود. به عنوان مثال:

میوه سیب، موز، انبه
ماشین ولوو، آئودی، تویوتا

وقتی یک شیء از یک کلاس ایجاد می‌کنید، تمام متغیرها و توابع تعریف‌شده در آن کلاس را به ارث می‌برد.

در فصل‌های بعدی، شما یاد خواهید گرفت که چگونه:

  • یک کلاس ایجاد کنید
  • اشیاء ایجاد کنید
  • به اعضای کلاس دسترسی پیدا کنید
  • و خیلی چیزهای دیگر

تکرارکننده‌ها در پایتون

تکرارکننده یک شیء است که شامل تعداد قابل شمارش از مقادیر است.

مدیریت خطاها در پایتون (Try Except)

بلوک try به شما امکان می‌دهد یک بلوک کد را برای بررسی خطاها آزمایش کنید.

بلوک except برای مدیریت خطاها استفاده می‌شود.

بلوک else به شما امکان می‌دهد کدی را اجرا کنید که در صورت نبود خطا اجرا شود.

بلوک finally کدی را اجرا می‌کند، صرف‌نظر از اینکه بلوک try خطا ایجاد کند یا خیر.

مدیریت استثناها

هنگامی که خطایی رخ می‌دهد (که به آن استثنا می‌گوییم)، پایتون معمولاً برنامه را متوقف کرده و پیغام خطا تولید می‌کند. این استثناها را می‌توان با استفاده از دستور try مدیریت کرد:

try_except.py

اگر بلوک try خطایی ایجاد کند، بلوک except اجرا می‌شود. بدون استفاده از try، برنامه متوقف شده و خطا نمایش داده می‌شود:

مدیریت فایل‌ها در پایتون

مدیریت فایل‌ها بخش مهمی از هر برنامه کاربردی وب است. پایتون توابعی برای ایجاد، خواندن، به‌روزرسانی و حذف فایل‌ها ارائه می‌دهد.

تابع open()

تابع کلیدی برای کار با فایل‌ها در پایتون، تابع open() است. این تابع دو پارامتر می‌گیرد: نام فایل و حالت (mode).

چهار حالت مختلف برای باز کردن فایل وجود دارد:

  • "r" - خواندن (Read) - مقدار پیش‌فرض. فایل را برای خواندن باز می‌کند، اگر فایل وجود نداشته باشد خطا می‌دهد.
  • "a" - افزودن (Append) - فایل را برای افزودن باز می‌کند، اگر فایل وجود نداشته باشد آن را ایجاد می‌کند.
  • "w" - نوشتن (Write) - فایل را برای نوشتن باز می‌کند، اگر فایل وجود نداشته باشد آن را ایجاد می‌کند.
  • "x" - ایجاد (Create) - فایل مشخص‌شده را ایجاد می‌کند، اگر فایل از قبل وجود داشته باشد خطا می‌دهد.

همچنین می‌توانید مشخص کنید که فایل به‌صورت متنی یا باینری مدیریت شود:

  • "t" - متنی (Text) - مقدار پیش‌فرض. حالت متنی.
  • "b" - باینری (Binary) - حالت باینری (مثلاً برای تصاویر).

توابع داخلی پایتون

پایتون مجموعه‌ای از توابع داخلی را ارائه می‌دهد که بدون نیاز به تعریف می‌توانید از آن‌ها استفاده کنید. این توابع برای انجام وظایف رایج در برنامه‌نویسی بسیار مفید هستند.

لیست توابع داخلی

در زیر فهرستی از توابع داخلی پایتون به همراه توضیح مختصر هر یک آورده شده است:

  • abs() - مقدار مطلق یک عدد را برمی‌گرداند.
  • all() - اگر همه آیتم‌های یک شیء قابل پیمایش درست باشند، True برمی‌گرداند.
  • any() - اگر حداقل یکی از آیتم‌های یک شیء قابل پیمایش درست باشد، True برمی‌گرداند.
  • ascii() - نسخه قابل خواندن یک شیء را برمی‌گرداند و کاراکترهای غیر ASCII را با کاراکترهای فرار جایگزین می‌کند.
  • bin() - نسخه باینری یک عدد را برمی‌گرداند.
  • bool() - مقدار بولی یک شیء را برمی‌گرداند.
  • bytearray() - آرایه‌ای از بایت‌ها را برمی‌گرداند.
  • bytes() - یک شیء بایت را برمی‌گرداند.
  • callable() - اگر شیء مشخص‌شده قابل فراخوانی باشد، True برمی‌گرداند.
  • chr() - کاراکتری را از کد یونیکد مشخص‌شده برمی‌گرداند.
  • classmethod() - یک متد را به متد کلاسی تبدیل می‌کند.
  • compile() - کد منبع مشخص‌شده را به یک شیء قابل اجرا تبدیل می‌کند.
  • complex() - یک عدد مختلط را برمی‌گرداند.
  • delattr() - ویژگی مشخص‌شده (ویژگی یا متد) را از یک شیء حذف می‌کند.
  • dict() - یک دیکشنری (آرایه) را برمی‌گرداند.
  • dir() - لیستی از ویژگی‌ها و متدهای یک شیء را برمی‌گرداند.
  • divmod() - خارج قسمت و باقی‌مانده تقسیم دو عدد را برمی‌گرداند.
  • enumerate() - یک مجموعه (مثل تاپل) را به یک شیء شمارش‌پذیر تبدیل می‌کند.
  • eval() - یک عبارت را ارزیابی و اجرا می‌کند.
  • exec() - کد یا شیء مشخص‌شده را اجرا می‌کند.
  • filter() - از یک تابع فیلتر برای حذف آیتم‌های یک شیء قابل پیمایش استفاده می‌کند.
  • float() - یک عدد اعشاری را برمی‌گرداند.
  • format() - یک مقدار را فرمت می‌کند.
  • frozenset() - یک شیء مجموعه غیرقابل تغییر (frozenset) را برمی‌گرداند.
  • getattr() - مقدار ویژگی مشخص‌شده (ویژگی یا متد) را برمی‌گرداند.
  • globals() - جدول نمادهای جهانی فعلی را به‌صورت دیکشنری برمی‌گرداند.
  • hasattr() - اگر شیء مشخص‌شده دارای ویژگی مشخص‌شده باشد، True برمی‌گرداند.
  • hash() - مقدار هش یک شیء را برمی‌گرداند.
  • help() - سیستم راهنمای داخلی را اجرا می‌کند.
  • hex() - یک عدد را به مقدار هگزادسیمال تبدیل می‌کند.
  • id() - شناسه (ID) یک شیء را برمی‌گرداند.
  • input() - امکان دریافت ورودی از کاربر را فراهم می‌کند.
  • int() - یک عدد صحیح را برمی‌گرداند.
  • isinstance() - اگر شیء مشخص‌شده نمونه‌ای از یک شیء خاص باشد، True برمی‌گرداند.
  • issubclass() - اگر کلاس مشخص‌شده زیرکلاسی از یک شیء خاص باشد، True برمی‌گرداند.
  • iter() - یک شیء تکرارپذیر را برمی‌گرداند.
  • len() - طول یک شیء را برمی‌گرداند.
  • list() - یک لیست را برمی‌گرداند.
  • locals() - دیکشنری به‌روزرسانی‌شده جدول نمادهای محلی فعلی را برمی‌گرداند.
  • map() - یک تابع را روی هر آیتم یک شیء تکرارپذیر اعمال می‌کند.
  • max() - بزرگ‌ترین آیتم در یک شیء تکرارپذیر را برمی‌گرداند.
  • memoryview() - یک شیء نمای حافظه را برمی‌گرداند.
  • min() - کوچک‌ترین آیتم در یک شیء تکرارپذیر را برمی‌گرداند.
  • next() - آیتم بعدی در یک شیء تکرارپذیر را برمی‌گرداند.
  • object() - یک شیء جدید را برمی‌گرداند.
  • oct() - یک عدد را به مقدار اکتال تبدیل می‌کند.
  • open() - یک فایل را باز کرده و یک شیء فایل را برمی‌گرداند.
  • ord() - کد یونیکد یک کاراکتر را برمی‌گرداند.
  • pow() - مقدار x به توان y را برمی‌گرداند.
  • print() - به دستگاه خروجی استاندارد چاپ می‌کند.
  • property() - یک ویژگی را دریافت، تنظیم یا حذف می‌کند.
  • range() - یک دنباله از اعداد را برمی‌گرداند (به‌طور پیش‌فرض از 0 شروع شده و با گام 1 افزایش می‌یابد).
  • repr() - نسخه قابل خواندن یک شیء را برمی‌گرداند.
  • reversed() - یک تکرارپذیر معکوس را برمی‌گرداند.
  • round() - یک عدد را گرد می‌کند.
  • set() - یک شیء مجموعه جدید را برمی‌گرداند.
  • setattr() - ویژگی (ویژگی یا متد) یک شیء را تنظیم می‌کند.
  • slice() - یک شیء برش را برمی‌گرداند.
  • sorted() - یک لیست مرتب‌شده را برمی‌گرداند.
  • staticmethod() - یک متد را به متد استاتیک تبدیل می‌کند.
  • str() - یک شیء رشته را برمی‌گرداند.
  • sum() - مجموع آیتم‌های یک تکرارپذیر را محاسبه می‌کند.
  • super() - یک شیء که نماینده کلاس والد است را برمی‌گرداند.
  • tuple() - یک تاپل را برمی‌گرداند.
  • type() - نوع یک شیء را برمی‌گرداند.
  • vars() - ویژگی __dict__ یک شیء را برمی‌گرداند.
  • zip() - یک تکرارپذیر از دو یا چند تکرارپذیر ایجاد می‌کند.

کلمات کلیدی پایتون

پایتون مجموعه‌ای از کلمات کلیدی دارد که کلمات رزروشده هستند و نمی‌توان از آن‌ها به‌عنوان نام متغیر، نام تابع یا هر شناسه دیگری استفاده کرد.

لیست کلمات کلیدی

در زیر فهرستی از کلمات کلیدی پایتون به همراه توضیح مختصر هر یک آورده شده است:

  • and - عملگر منطقی برای ترکیب شرط‌ها.
  • as - برای ایجاد نام مستعار (alias).
  • assert - برای اشکال‌زدایی (debugging).
  • break - برای خروج از یک حلقه.
  • class - برای تعریف یک کلاس.
  • continue - برای ادامه دادن به تکرار بعدی یک حلقه.
  • def - برای تعریف یک تابع.
  • del - برای حذف یک شیء.
  • elif - در دستورات شرطی استفاده می‌شود، معادل else if.
  • else - در دستورات شرطی استفاده می‌شود.
  • except - با استثناها استفاده می‌شود، مشخص می‌کند در صورت بروز استثنا چه کاری انجام شود.
  • False - مقدار بولی، نتیجه عملیات مقایسه.
  • finally - با استثناها استفاده می‌شود، بلوکی از کد که در هر صورت اجرا می‌شود.
  • for - برای ایجاد یک حلقه for.
  • from - برای وارد کردن بخش‌های خاصی از یک ماژول.
  • global - برای اعلان یک متغیر جهانی.
  • if - برای ایجاد یک دستور شرطی.
  • import - برای وارد کردن یک ماژول.
  • in - برای بررسی وجود یک مقدار در لیست، تاپل و غیره.
  • is - برای آزمایش برابری دو متغیر.
  • lambda - برای ایجاد یک تابع ناشناس.
  • None - نشان‌دهنده مقدار تهی (null).
  • nonlocal - برای اعلان یک متغیر غیرمحلی.
  • not - عملگر منطقی برای نفی.
  • or - عملگر منطقی برای انتخاب.
  • pass - دستور خنثی، عملی انجام نمی‌دهد.
  • raise - برای ایجاد یک استثنا.
  • return - برای خروج از یک تابع و بازگرداندن یک مقدار.
  • True - مقدار بولی، نتیجه عملیات مقایسه.
  • try - برای ایجاد یک دستور try...except.
  • while - برای ایجاد یک حلقه while.
  • with - برای ساده‌سازی مدیریت استثناها.
  • yield - برای بازگرداندن لیستی از مقادیر از یک ژنراتور.

استثناهای داخلی پایتون

پایتون مجموعه‌ای از استثناهای داخلی دارد که معمولاً در هنگام بروز خطاها ایجاد می‌شوند. این استثناها به شما کمک می‌کنند تا خطاها را شناسایی و مدیریت کنید.

لیست استثناهای داخلی

در زیر فهرستی از استثناهای داخلی پایتون به همراه توضیح مختصر هر یک آورده شده است:

  • ArithmeticError - هنگامی که خطایی در محاسبات عددی رخ می‌دهد، ایجاد می‌شود.
  • AssertionError - هنگامی که یک دستور assert ناموفق باشد، ایجاد می‌شود.
  • AttributeError - هنگامی که ارجاع یا تخصیص ویژگی ناموفق باشد، ایجاد می‌شود.
  • Exception - کلاس پایه برای همه استثناها.
  • EOFError - هنگامی که متد input() به شرایط "پایان فایل" (EOF) برخورد کند، ایجاد می‌شود.
  • FloatingPointError - هنگامی که یک محاسبه اعشاری ناموفق باشد، ایجاد می‌شود.
  • GeneratorExit - هنگامی که یک ژنراتور با متد close() بسته شود، ایجاد می‌شود.
  • ImportError - هنگامی که ماژول واردشده وجود نداشته باشد، ایجاد می‌شود.
  • IndentationError - هنگامی که تورفتگی (indentation) نادرست باشد، ایجاد می‌شود.
  • IndexError - هنگامی که اندیس یک دنباله وجود نداشته باشد، ایجاد می‌شود.
  • KeyError - هنگامی که کلیدی در دیکشنری وجود نداشته باشد، ایجاد می‌شود.
  • KeyboardInterrupt - هنگامی که کاربر کلیدهای Ctrl+C، Ctrl+Z یا Delete را فشار دهد، ایجاد می‌شود.
  • LookupError - هنگامی که خطاهای ایجادشده یافت نشوند، ایجاد می‌شود.
  • MemoryError - هنگامی که برنامه با کمبود حافظه مواجه شود، ایجاد می‌شود.
  • NameError - هنگامی که یک متغیر وجود نداشته باشد، ایجاد می‌شود.

  • OverflowError - هنگامی که نتیجه یک محاسبه عددی بیش از حد بزرگ باشد، ایجاد می‌شود.
  • ReferenceError - هنگامی که یک شیء ارجاع ضعیف وجود نداشته باشد، ایجاد می‌شود.
  • RuntimeError - هنگامی که خطایی رخ دهد که به هیچ استثنای خاصی تعلق ندارد، ایجاد می‌شود.
  • StopIteration - هنگامی که متد next() یک تکرارکننده دیگر مقداری نداشته باشد، ایجاد می‌شود.
  • SyntaxError - هنگامی که خطای نحوی (syntax) رخ دهد، ایجاد می‌شود.
  • TabError - هنگامی که تورفتگی شامل تب یا فاصله باشد، ایجاد می‌شود.
  • SystemError - هنگامی که خطای سیستمی رخ دهد، ایجاد می‌شود.
  • SystemExit - هنگامی که تابع sys.exit() فراخوانی شود، ایجاد می‌شود.
  • TypeError - هنگامی که دو نوع مختلف ترکیب شوند، ایجاد می‌شود.
  • UnboundLocalError - هنگامی که یک متغیر محلی قبل از تخصیص ارجاع داده شود، ایجاد می‌شود.

پی‌آی‌پی یا pip

pip چیست؟

Pip یک مدیر بسته برای بسته‌ها یا ماژول‌های پایتون است.

نکته: اگر از پایتون نسخه ۳.۴ یا بالاتر استفاده می‌کنید، پی‌آی‌پی به‌صورت پیش‌فرض نصب شده است.

بسته چیست؟

یک بسته شامل تمام فایل‌های موردنیاز برای یک ماژول است. ماژول‌ها کتابخانه‌های کد پایتون هستند که می‌توانید در پروژه خود استفاده کنید.

بررسی نصب بودن پی‌آی‌پی

به خط فرمان خود بروید، به پوشه اسکریپت‌های پایتون (Python's script directory) بروید و دستور زیر را وارد کنید:

pip --version

این دستور نسخه نصب‌شده پی‌آی‌پی را نمایش می‌دهد، مثلاً:

pip 23.0.1

نصب پی‌آی‌پی

اگر پی‌آی‌پی نصب نیست، می‌توانید آن را از این صفحه دانلود و نصب کنید: https://pypi.org/project/pip/

دانلود یک بسته

دانلود یک بسته بسیار ساده است. خط فرمان را باز کنید و به پی‌آی‌پی دستور دهید بسته موردنظر را دانلود کند.

به پوشه اسکریپت‌های پایتون بروید و دستور زیر را وارد کنید:

pip install Package_name

اکنون بسته دانلود و نصب شده است!

استفاده از یک بسته

پس از نصب بسته، آماده استفاده است. بسته را در پروژه خود وارد کنید.

camelcase_example.py

یافتن بسته‌ها

برای یافتن بسته‌های بیشتر، به https://pypi.org/ مراجعه کنید.

حذف یک بسته

برای حذف یک بسته از دستور uninstall استفاده کنید:

pip uninstall Package_name

فهرست بسته‌ها

از دستور list برای فهرست کردن تمام بسته‌های نصب‌شده در سیستم خود استفاده کنید:

pip list
